Files Viewer

Objectif

Permettre à tout utilisateur autorisé à voir un skill de consulter en lecture seule les fichiers d'une version (fichier principal, fichiers texte auxiliaires, fichiers binaires) sans passer par la page d'édition. Évalue le contenu avant installation ou téléchargement.

Où le trouver

Sur la page détail d'un skill (/skill/<id>), un onglet « Files » est désormais affiché entre Overview et Install. Ordre final des onglets :

Overview · Files · Install · Comments · Versions

L'onglet est visible pour tout utilisateur ayant accès à la page détail — la visibilité (public / private / team) reste filtrée côté backend par GET /api/versions/<id>.

Fonctionnalités

Élément Comportement
Sélecteur de version Liste toutes les versions du skill, présélectionne current_version. Affiche le préfixe 8 caractères de content_hash_sha256 pour traçabilité.
Arborescence Vue lecture seule de tous les fichiers de la version (texte + binaires). Aucune action de création / renommage / suppression / upload n'est rendue.
Viewer Markdown SKILL.md, .md, .mdc sont rendus en HTML stylé.
Viewer code .json, .yaml, .yml, .sh, .py, .ts, .tsx, .js, .jsx, .sql, .html, .css sont affichés avec coloration syntaxique.
Viewer texte brut Toute autre extension texte s'affiche dans un bloc monospace.
Viewer binaire Images (PNG/JPG/GIF/WEBP/SVG), PDF, audio (MP3/WAV/OGG…), vidéo (MP4/WEBM/MOV…) sont prévisualisés en ligne via data-URI.
Carte info binaire Pour les binaires non prévisualisables (.zip, .tar, etc.), affiche nom + taille + bouton Download.
Download par fichier Bouton Download disponible sur chaque fichier (texte ou binaire).
Download ZIP Raccourci vers /api/skills/<id>/download?tool=portal&version_id=<id> pour récupérer toute la version.

Garanties de lecture seule

  • L'onglet n'appelle aucun verb HTTP autre que GET — pas de useMutation, pas de POST/PATCH/PUT/DELETE accessible.
  • Le composant SkillFileTree est monté avec readOnly={true} : les actions d'édition (rename, delete, create, upload) ne sont pas rendues dans le DOM.
  • Un éditeur autorisé continue d'utiliser le bouton Edit (dans l'en-tête de la page détail) pour accéder à l'interface d'édition /skill/<id>/edit.

Performance

Le composant SkillFilesTab est chargé via React.lazy + Suspense : le chunk n'est téléchargé que lorsque l'utilisateur clique pour la première fois sur l'onglet « Files ». Les versions déjà consultées sont mises en cache par TanStack Query.

Internationalisation

L'onglet est traduit dans les 6 langues supportées : fr, en, de, es, zh, ja. Clés i18n :

  • skillDetail.tab.files
  • skillFiles.version.label, skillFiles.version.hash
  • skillFiles.empty, skillFiles.loading, skillFiles.error
  • skillFiles.action.copyPath, skillFiles.action.download, skillFiles.action.downloadZip

Accessibilité

  • Navigation clavier complète (Tab → version selector → tree → viewer actions).
  • Anneau de focus visible sur tous les éléments interactifs.
  • Zones aria-live="polite" sur le viewer pour annoncer les changements d'état (loading, error).
  • Tree exposé avec role="tree" via SkillFileTree.
